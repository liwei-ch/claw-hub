
# A股自动交易系统设计方案V1.0
## 一、设计原则
本方案100%对齐用户V2.0交易体系规则，基于「大资金体量适配+策略可迭代扩展」核心诉求，遵循三大设计原则：
1. **风控优先原则**：所有交易请求必经风控层校验，违规交易100%拦截，完全杜绝超额风险
2. **可扩展原则**：策略、因子、交易品类支持热插拔扩展，无需重构核心架构
3. **高可用原则**：全链路冗余设计，交易时段可用性≥99.99%，信号延迟≤2秒

---
## 二、总体架构设计（分层架构）
```
┌──────────────────────────────────────────────────────────┐
│  通知运营层：飞书通知、运营后台、数据可视化、绩效统计      │
├──────────────────────────────────────────────────────────┤
│  交易执行层：同花顺智能交易API对接、订单管理、拆单算法      │
├──────────────────────────────────────────────────────────┤
│  策略核心层：V2.0内置策略、因子引擎、信号生成、回测引擎      │
├──────────────────────────────────────────────────────────┤
│  风控拦截层：仓位校验、回撤校验、异常校验、应急开关          │
├──────────────────────────────────────────────────────────┤
│  数据采集层：聚宽JQData对接、行情数据、特色数据、数据清洗      │
└──────────────────────────────────────────────────────────┘
```
### 各层核心职责
1. **数据采集层**：负责全量数据的拉取、清洗、标准化，向上层提供统一的数据接口
2. **风控拦截层**：所有交易请求的必经关卡，100%前置校验所有风控规则，不符合规则直接拦截
3. **策略核心层**：负责策略逻辑执行、因子计算、信号生成、历史回测，是系统核心大脑
4. **交易执行层**：负责对接交易接口，执行订单、管理交易状态、实现拆单算法减少大资金冲击成本
5. **通知运营层**：负责消息推送、数据展示、策略配置，提供用户交互入口

## 三、核心模块详细设计
### 🔹 1. 数据采集模块
#### 核心接口设计
| 接口名 | 入参 | 出参 | 调用频率 |
|--------|------|------|----------|
| get_realtime_quota | 股票代码列表 | 实时价格、涨跌幅、成交量、均线数据 | 每分钟1次 |
| get_market_sentiment | 无 | 涨跌家数、涨跌停数、炸板率、北向资金流向 | 每分钟1次 |
| get_stock_factor | 股票代码、因子列表 | 指定因子的计算结果 | 每日盘后1次 |
| get_plate_data | 板块代码 | 板块涨跌幅、资金流向、领涨龙头 | 每分钟1次 |
#### 异常处理
- 数据源请求失败自动重试3次，每次间隔1秒，仍失败则触发飞书告警，使用上一次有效数据
- 数据校验不合法自动丢弃，避免脏数据导致决策错误

---
### 🔹 2. 风控拦截模块（大资金核心保障）
#### 校验流程（所有交易请求必经）
```
交易请求 → 仓位规则校验 → 回撤规则校验 → 异常规则校验 → 通过/拦截
```
#### 内置校验规则（完全对齐V2.0体系）
| 校验规则 | 逻辑实现 | 拦截动作 |
|----------|----------|----------|
| 总仓位约束 | 总仓位≤100%，趋势/短线仓位严格5:5 | 超仓位直接拦截 |
| 单票/单行业约束 | 单票仓位≤2层，单行业总仓位≤2层，总持仓≤5只 | 超约束直接拦截 |
| 回撤约束 | 单票回撤≤30%，单日总账户亏损≤2%，单票单日跌幅≤8% | 触发自动止损/减仓 |
| 异常交易约束 | 大盘单日跌幅≥5%禁止开仓，10分钟内交易次数≤5次 | 异常情况拦截交易并告警 |
#### 可配置能力
所有风控阈值支持通过配置文件动态修改，无需重启系统。

---
### 🔹 3. 策略核心模块
#### 3.1 内置V2.0策略实现
| 逻辑模块 | 实现细节 |
|----------|----------|
| 趋势判断 | 以10日均线为核心，收盘价站稳10均线判定趋势向上，跌破判定趋势走弱 |
| 选股逻辑 | 优先过滤8大主线赛道标的，优先选择行业/细分龙头标的 |
| 信号生成 | 买入信号：板块效应+弱转强拐点+支撑位K线信号+仓位空位<br>卖出信号：指标转弱+破位+压力位不过+形态拐点 |
| 仓位分配 | 趋势票单票1-2层，短线票0.5-1层，自动计算可用仓位分配 |
#### 3.2 因子引擎设计
- 内置聚宽2000+量化因子适配器，可通过配置直接调用
- 支持自定义因子热加载：用户编写Python函数放在指定目录，自动加载生效
- 支持多因子组合权重配置，通过配置文件调整因子占比
#### 3.3 回测引擎设计
- 支持日线/分钟级历史数据回测，回测时间范围可自定义
- 自动计算收益率、最大回撤、夏普比率、胜率、盈亏比等绩效指标
- 自动生成可视化回测报告，输出交易明细、持仓变化曲线

---
### 🔹 4. 交易执行模块
#### 核心功能实现
| 功能 | 实现细节 |
|------|----------|
| 基础交易 | 对接同花顺API，实现买卖、撤单、查持仓、查成交、打新等功能 |
| 拆单算法 | 大资金订单自动拆分为多笔≤10万的小单，随机间隔下单，避免冲击成本 |
| 订单管理 | 实时跟踪订单状态，失败订单自动重试2次，仍失败触发告警 |
| 模式切换 | 支持全自动/半自动/手动三种模式切换，切换即时生效无需重启 |
#### 交易日志
所有交易操作全量记录，包含：触发信号、订单参数、执行结果、耗时，保存周期≥3年，可追溯可审计。

---
### 🔹 5. 通知运营模块
#### 飞书通知事件
| 事件类型 | 推送内容 | 推送时机 |
|----------|----------|----------|
| 交易信号 | 信号类型、标的名称、建议操作、仓位 | 信号生成时推送 |
| 交易执行 | 订单执行结果、成交价格、成交量 | 订单成交时推送 |
| 风控告警 | 拦截原因、风险点、建议操作 | 风控拦截/触发阈值时推送 |
| 日报/周报 | 当日/当周持仓、盈亏、交易汇总、绩效指标 | 每日盘后/每周五盘后推送 |
#### 运营后台
基于vn.py自带的可视化后台，实现：
- 实时持仓、交易记录、信号历史查询
- 策略参数、因子权重、风控阈值配置
- 回测报告、绩效统计查看

## 四、全流程数据流向
```
1. 数据采集层每分钟拉取聚宽JQData行情、资金、情绪、板块数据
2. 数据清洗标准化后送入策略核心层
3. 策略核心层执行因子计算、逻辑判断，生成交易信号
4. 交易信号送入风控拦截层，逐层校验所有风控规则
5. 校验通过的信号送入交易执行层
6. 交易执行层根据订单大小自动拆单，调用同花顺API提交交易
7. 交易结果返回给通知运营层，推送飞书通知给用户
8. 所有日志、数据存入数据库，用于后续回测、绩效统计
```

## 五、部署方案

| 配置项 | 最低要求 | 推荐配置 |

| CPU | 4核 | 8核 |

- 交易时段7*24小时运行，非交易时段自动备份数据

3. 导入V2.0策略规则、风控阈值配置
4. 模拟盘测试所有功能
5. 小资金实盘测试
6. 全量上线

## 七、运维与风险预案
### 日常运维
- 每日盘后自动备份数据、日志
- 每周自动运行健康检查，检查接口可用性、系统状态
- 每月自动生成系统运行报告，统计可用性、交易成功率
### 风险预案
| 风险场景 | 应对方案 |
|----------|----------|
| 聚宽数据源故障 | 自动切换到备用数据源（Tushare），同时推送告警 |
| 同花顺交易接口故障 | 自动暂停交易，推送告警，用户可手动操作 |
| 服务器宕机 | 自动切换到备用节点，不影响交易 |
| 策略逻辑异常 | 触发异常交易监控，自动暂停交易，推送告警 |
| 极端行情（大盘暴跌） | 自动触发异常风控规则，禁止新开仓，推送告警 |
