
# A股自动交易系统需求调研文档V1.0
## 📋 项目背景
本系统基于用户制定的《V2.0闭环交易体系》量身打造，服务于**大资金体量交易、策略持续迭代进化扩展决策因子**的核心诉求，实现交易全流程自动化、规则执行100%严格化、策略迭代快速化，完全规避人工交易的情绪风险、执行力不足问题。

---
## 🎯 核心诉求明确
### 1. 大资金体量适配
- 支持大资金交易场景，避免冲击成本、滑点过大
- 多层级强风控保障，杜绝异常交易、超额亏损
- 交易链路高稳定、高可用，避免交易信号漏发、错发导致的大额损失

### 2. 策略可进化扩展
- 支持灵活新增/修改决策因子（技术面/基本面/资金面/情绪面等）
- 支持策略逻辑快速迭代，新增规则无需重构系统
- 配套回测验证能力，新策略/新因子上线前可快速验证历史收益表现

---
## ✨ 功能需求清单
### 🔹 1. 数据采集模块
| 功能点 | 详细要求 |
|--------|----------|
| 行情数据采集 | 对接聚宽JQData，实时获取A股个股/板块/指数的行情数据：实时价格、均线（5/10/20日等）、成交量、涨跌幅、换手率等基础数据 |
| 特色数据采集 | 对接聚宽JQData，获取进阶特色数据：<br>1. 情绪类：涨跌家数、涨跌停数量、炸板率、赚钱效应、北向/南向资金流向<br>2. 资金类：个股主力资金流向、龙虎榜数据、机构持仓比例<br>3. 基本面类：PE/PB/ROE/净利润增速等财务指标、公告/研报/业绩预告等事件数据<br>4. 板块类：行业/概念板块涨跌幅排行、板块资金流向、领涨龙头识别 |
| 数据更新频率 | 日线级别数据每日盘后更新，日内交易信号数据每分钟更新一次（可配置） |
| 异常处理 | 数据源异常时自动重试+告警，避免数据缺失导致交易决策错误 |

---
### 🔹 2. 策略核心模块（100%对齐V2.0交易体系，支持扩展）
| 功能点 | 详细要求 |
|--------|----------|
| 基础策略逻辑（内置） | 完全内置V2.0交易体系所有规则+补充隐性交易习惯：<br>1. 趋势判断：10均线为核心，自动识别趋势向上/走弱/破位<br>2. 选股规则：优先8大主线赛道（光伏>固态电池>化工>AI应用>机器人>有色>半导体>军工）、优先行业/细分龙头<br>3. 信号识别：自动识别买入/卖出/加仓/减仓信号（K线形态、量能、资金、板块效应等）<br>4. 仓位控制：严格遵守「50%趋势+50%短线」总仓位结构、单票1-2层/单行业≤2层/总持仓≤5只的硬约束<br>5. 交易时段规则：全天任意时间信号触发即可执行交易<br>6. 黑名单规则：ST票、有减持公告的标的禁止开仓<br>7. 特殊时段规则：季报/年报披露期无需额外降仓/空仓，按正常规则执行 |
| 因子扩展能力 | 支持灵活新增自定义决策因子，无需重构代码：<br>1. 内置聚宽2000+量化因子库，可直接配置调用<br>2. 支持用户自定义编写因子逻辑，热更新上线<br>3. 支持多因子组合权重配置 |
| 策略迭代能力 | 支持多策略并行、策略热切换、策略版本管理，新策略上线无需停系统 |
| 回测验证功能 | 内置历史回测模块：<br>1. 支持用3-5年历史行情数据回测新策略/新因子的收益率、最大回撤、胜率、盈亏比等绩效指标<br>2. 自动生成回测报告，验证有效后再上线实盘 |

---
### 🔹 3. 交易执行模块
| 功能点 | 详细要求 |
|--------|----------|
| 接口对接 | 对接同花顺智能交易API，支持全券商交易 |
| 基础交易功能 | 自动执行买卖、撤单、查持仓、查成交、打新等所有常规交易操作 |
| 大资金适配功能 | 支持拆单算法：大金额订单自动拆分为多笔小单成交，避免冲击成本、减少滑点 |
| 交易模式切换 | 支持三种模式灵活切换：<br>1. 全自动模式：信号触发自动下单，无需人工干预<br>2. 半自动模式：信号触发后推送通知，人工确认后再执行交易<br>3. 手动模式：完全关闭自动交易，仅保留信号提醒功能 |
| 交易日志 | 全量记录所有交易操作日志、信号触发日志，可追溯可审计 |

---
### 🔹 4. 风控模块（大资金核心保障）
| 功能点 | 详细要求 |
|--------|----------|
| 仓位风控 | 实时监控所有仓位约束：<br>1. 总仓位不超过100%、趋势/短线仓位比例严格5:5<br>2. 单票仓位≤2层、单行业总仓位≤2层、总持仓数量≤5只<br>3. 违反约束的订单直接拦截，禁止提交 |
| 回撤风控 | 实时监控回撤：<br>1. 单票最大回撤≤30%，触发自动止损<br>2. 单日总账户亏损≥2%，自动暂停所有交易并告警<br>3. 单票单日跌幅≥8%，自动减仓1/2 |
| 异常风控 | 异常情况自动拦截交易：<br>1. 大盘单日跌幅≥5%，禁止新开仓<br>2. 行情数据源/交易接口异常，自动暂停交易并告警<br>3. 短时间内连续触发多次交易信号，自动暂停并人工校验<br>4. 大额交易阈值触发时，自动暂停交易推送人工确认，确认后再执行<br>5. 任何异常/极端行情场景下，优先暂停交易等待人工处理，不自动平仓/止损 |
| 应急开关 | 支持一键暂停所有自动交易、一键清仓等应急操作 |

---
### 🔹 5. 通知与运营模块
| 功能点 | 详细要求 |
|--------|----------|
| 实时通知 | 所有事件通过飞书推送给用户：<br>1. 交易信号触发、交易执行结果通知<br>2. 风控告警、异常情况通知<br>3. 每日盘后持仓报告、交易汇总报告<br>4. 每周/每月策略绩效报告 |
| 运营后台 | 简单可视化界面支持：<br>1. 查看实时持仓、交易记录、信号历史<br>2. 配置策略参数、因子权重、风控阈值<br>3. 查看回测报告、绩效统计 |

## 🛡️ 非功能需求
1. **稳定性**：系统可用率≥99.99%，全年 downtime 不超过52分钟，交易时段无故障
2. **性能**：行情数据更新延迟≤1秒，交易信号触发到下单延迟≤2秒，可支撑单日最多100笔交易
3. **安全性**：所有接口密钥、交易密码加密存储，无泄露风险；所有操作日志不可篡改
4. **可扩展性**：支持新增交易品类（港股/期货/可转债等）、新增策略、新增因子无需重构系统架构
5. **可维护性**：代码结构清晰，文档完善，用户可自行修改简单逻辑
