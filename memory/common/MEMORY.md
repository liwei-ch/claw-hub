## Cross-Persona Lessons（跨角色重要教训）

- 开发规范铁律：所有项目开发必须遵循对应项目根目录的 AGENTS.md 规范；无规范时先补规范再开发，禁止随意扩展接口与参数。
- 记忆体系优化经验：关键指令与规则需要及时沉淀到长期记忆，避免依赖短期上下文；系统重启时要有明确的「启动流程」保证状态一致性。
- 自我进化日志结构：每日进化建议采用固定结构（信息收集 → 用户互动分析 → 问题/机会识别 → 已完成改进 → 待继续改进 → 今日收获 → 核心结论），便于跨日对比和长期追踪。
- 稳定优先于盲目快迭代：在用户已有较好体验和正向反馈时，优先保证稳定运行和计划内改进，而不是频繁大改核心流程。
- 错误处理三步法：复盘根因 → 制定流程级防重复方案 → 在记忆与文档中双重沉淀，避免只停留在这一次修复。
