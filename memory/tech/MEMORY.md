## Patterns & Lessons

- 记忆丢失/偏差问题：
  - 已通过关键指令实时持久化、启动流程校验等方式进行优化；之后在新增功能时要优先考虑「记忆写入点」和「重启恢复路径」。
- 配置修改规范：
  - 修改前必须备份原始配置，核对关键参数（尤其是 appId/appSecret 这类成对字段），修改后再验证关键功能链路是否正常。
- 长任务与超时：
  - 任何可能长时间运行的任务都应设计合理超时时间和超时后的处理策略（自动终止/告警/可重试），避免长期占用资源或卡住进程。
- 数据校验思路（技术视角）：
  - 错误的数据比没有数据危害更大，要优先考虑接口字段含义、数据范围合理性和多源校验机制，再谈上层分析与展示。
